###什么是控制指令？
　　本质上，**控制指令**是一条由用户（或您的接口）主动发送到开发板上的HTTP访问，用于通知开发板执行对应的操作。
　　简单的说，开发板通过HTTP主动向您的接口推送的消息，称为“**推送消息**”；由您的接口（或浏览器）通过HTTP主动向开发板推送的消息，称为“**控制指令**”。
```seq
您的接口-开发板:&amp;rarr;　　　　　控制指令　　　　　&amp;rarr;
```
###控制指令的示例
　　以获取**开发板当前状态(stat)**的指令为例，说明控制指令的构成：
  ```
  http://192.168.7.170:38585/ctrl?token=3f4bffa77257d243875d0a5a80635934&amp;cmd=stat
  ```
 　　`http://192.168.7.170:38585/ctrl`这是控制指令的固定开头。其中`192.168.7.170`是开发板的当前IP地址，实际使用中需要替换成您的开发板真实的IP地址。
 　　`:38585`是开发板固定的WEB端口号。
 　　`?token=3f4bffa77257d243875d0a5a80635934`**token**所有控制指令的所必须的一个参数，具体**token**的获取方法，请参阅**安全模型与token机制**的相关章节。
 　　`&amp;cmd=stat`**cmd**是控制指令所必须的另一个参数。用于指明开发板具体执行什么动作。
   
 　　这条**控制指令**的作用是，获取开发板当前的各项状态，您可以尝试在浏览器上输入这个URL，之后开发板会以JSON格式返回当前自身的状态信息，示例如下：
   
 ``` 
 {
  &quot;devId&quot;: &quot;498060912345&quot;,　　//当前开发板唯一ＩＤ
  &quot;schRstHour&quot;: 4,			//开发板每日定时重启时间，默认凌晨4点
  &quot;pingSec&quot;: 110,			//开发板主动推送PING消息（998）的间隔秒数
  &quot;wifi&quot;: {
    &quot;ssid&quot;: &quot;mywifi&quot;,		//当前连入的WIFI热点名
    &quot;ip&quot;: &quot;192.168.7.170&quot;,	//当前开发板的IP地址
    &quot;dbm&quot;: 21				//当前WIFI的信号强度
  },
  &quot;slot&quot;: {
    &quot;slot1_sta&quot;: &quot;ERR&quot;,		//卡槽1通信模组状态：异常（不可识别）
    &quot;slot2_sta&quot;: &quot;POWON&quot;,	//卡槽2通信模组状态：正常开机
    &quot;sim2_op&quot;: &quot;T&quot;,			//卡槽2中SIM卡所属运营商：中国电信
    &quot;sim2_dbm&quot;: &quot;99&quot;,		//卡槽2中SIM卡的信号强度：99%
    &quot;sim2iccId&quot;: &quot;89860321745322212345&quot;,	//卡槽2中SIM卡的ICCID
    &quot;sim2_sta&quot;: &quot;OK&quot;		 //卡槽2中SIM卡的当前状态：正常就绪
  },
  &quot;time&quot;:&quot;220618101522&quot;		//开发板当前时间yymmddhhmmss
} 

 ```
 　　如果您使用TCP接口与设备通信，可以用下面的消息获取设备状态：
   ```
   {&quot;cmd&quot;:&quot;stat&quot;, &quot;tid&quot;:&quot;1234&quot;}\x11\x12
   ```
 　　注意：TCP接口通信时，每一条消息最后都必须附加2个字节的结束符，用于表示消息的结束。2个字节的结束符必须是0x11 0x12。即结束符第一个字节是0x11，第二个字节是0x12。
   ```
   注意：
   0x11 0x12 分别代表2个字节，不要写成字符串。
   ```
